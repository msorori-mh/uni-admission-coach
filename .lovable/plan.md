

# خطة: إشعار تفصيلي للطالب عند رفض الدفع بسبب عدم تطابق البيانات

## ملخص
تحسين رسالة الرفض التلقائية لتتضمن تفاصيل دقيقة عن عدم التطابق (اسم المستلم المستخرج من السند مقابل اسم المستلم المتوقع)، بحيث يصل الطالب إشعار واضح يفهم منه سبب الرفض بالتحديد.

## الوضع الحالي
- يوجد trigger في قاعدة البيانات (`notify_student_on_payment_review`) يُرسل إشعاراً للطالب عند رفض الدفع ويتضمن `admin_notes` كسبب
- عند عدم التطابق، يتم تعبئة `adminNotes` تلقائياً بنص عام: "بيانات المستلم في السند غير مطابقة"
- **لا تحتاج تغييرات في قاعدة البيانات** — الـ trigger يعمل بشكل صحيح

## التغيير المطلوب

### تعديل واحد في `src/pages/admin/AdminPayments.tsx`
تحسين نص الرفض التلقائي (سطر 106) ليتضمن التفاصيل:

**قبل:**
```
"بيانات المستلم في السند غير مطابقة لبيانات طريقة الدفع المسجلة."
```

**بعد:**
```
"تم رفض طلب الدفع: بيانات المستلم غير مطابقة.
اسم المستلم في السند: [الاسم المستخرج]
اسم المستلم المتوقع: [الاسم من طريقة الدفع]
يرجى التأكد من التحويل إلى الحساب الصحيح وإعادة المحاولة."
```

- يتم بناء النص ديناميكياً باستخدام `data.recipient_name` و `expectedAccountName`
- إذا لم يُستخرج اسم المستلم، يُذكر "غير واضح" بدلاً منه
- الـ trigger الموجود سيرسل هذا النص التفصيلي كاملاً في الإشعار تلقائياً

## التفاصيل التقنية
- ملف واحد فقط يتأثر: `src/pages/admin/AdminPayments.tsx`
- تعديل دالة `analyzeReceipt` لتمرير `expectedAccountName` وبناء رسالة رفض مفصّلة
- لا حاجة لتعديل الـ trigger أو Edge Function
